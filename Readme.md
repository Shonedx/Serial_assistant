## **上位机** 
1. 搭配电机控制使用 通常使用串口通信
2. 支持自定义协议头
3. 通过协议头来分辨命令格式 可解析数据来调整对应的参数
4. 支持电机pid参数实时调整
5. 通过将需要调整的数据整合成一个包发送给接收端
6. ~~需要给接收端提供解包的api 可用c语言或者python来写~~
7. 需要使用python的图形库或者第三方图形库 程序需要有UI 用户可以通过拖动ui组件来调整参数，或者选择输入详细的数据 图形库要轻量美观 ui形式要简洁
8. 需要实现串口助手的基本功能，比如原样打印下位机/单片机发送来的数据
9. 下位机端可以切换发送模式为hex/文本模式 文本模式可以调整GBK格式或者UTF-8格式
10. 上位机端可以切换接收为hex/文本模式 文本模式可以调整GBK格式或者UTF-8格式
11. 界面需要有串口配置 比如 波特率 串口号 数据位 停止位 校验位 以及关闭串口打开串口的操作
12. 上位机要有发送区域和接收区域显示
13. pid 等参数的调整需要放到另一个窗口里，通过侧边栏的选项来切换窗口
14. 窗口切换的图标需要简洁优雅 且在侧边栏的比较小的位置 中文字体要用楷书 且串口配置的功能要放在串口助手窗口里 
15. 图标的大小要随着窗口变化而变化 同时图标大小要合适 不能太小不能太大
16. 关于自定义参数格式这一块 我需要你把自定义参数窗口设置成可自行添加发送数据格式 对应数据的名称 
发送模式：1. 自动模式：比如多少间隔发送一次 或者一次性发送多少次 2.手动模式：用户手动点击发送来发送对应数据
组件设计：把自定义参数格式设计成添加对应的组件，组件含有可拖动的滑杆，手动输入的文本栏(不能太大，大小合适，可以通过用户输入数据的长短变化，但是输入完后切换回常态的显示形式，对于超出显示范围的数据以滚动形式显示) 
该窗口也需要有个显示下位机发来的文本的位置，以便于用户查看调整
17.  手动模式的时候摇杆调整自动发送数据的bug修一下
18. 用户可以手动拖动对应得区域得大小，同时我建议显示窗口大一点
19. 自动发送模式得时间可以自己调整参数和单位
20. 原来串口窗口得发送栏目要保留且可以调整大小 串口窗口要可以调整串口配置 比如 波特率 串口号 数据位 停止位 校验位 以及关闭串口打开串口的操作
21. 参数设置栏目对于每个组件要可以调整滑杆得最大值和最小值
22. 增加一个能发送特定格式文本得组件
23. 提供一个串口助手的文本显示框的颜色调整的功能
24. 增加清除上位机接收区域和发送区域信息的功能
